首先，软件的设计框架从大到小等级是：方案-》流程-》节点，软件运行期间只会维护一个方案，单个
方案可以创建多个流程，而每个流程要执行的动作（像通讯、拍照、检测识别等等）都会被抽象为一个个节点
，多个模块按照着顺序排列执行，从而完成一个流程的功能。软件运行一个检测方案就能够将多个检测流程同
时/或者按照期待的顺序执行。

1. 本目录用于放置各种类型的节点

2. 每个新的节点类型定义参照NodeDemo目录实现

3. 本目录下还会根据功能不同细分为几个大类，暂定为“光源”、“相机”、“传统算法”、“AI”、“PLC类节点”……
